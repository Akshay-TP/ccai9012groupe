# Data Documentation

This folder contains static inputs, reproducible raw downloads, and source metadata used by the transport inequality workflow.

The ingestion stage (`src/01_fetch_data.py`) populates `data/raw/` so downstream scripts can execute without repeated API calls during every stage.

## Directory Contents

- `population_by_district.csv`
	- curated district control totals (population and area)
- `raw/`
	- API payloads, derived point tables, and geospatial source files generated by fetch stage

## Data Pipeline Role

These datasets support four model layers:

1. Service supply layer (stops, routes, stop-route mappings)
2. Spatial context layer (district boundaries)
3. Physical access layer (topography + ramp proxy points)
4. Population layer (district controls + fine population metadata)

## Key Local Schema Notes

### `population_by_district.csv`

Required columns:

- `district`
- `population`
- `area_km2`

Purpose:

- Provides district control totals for per-capita metrics.
- Anchors micro-grid population redistribution in `src/03_compute_accessibility.py`.

### `raw/hk_topography_points.csv`

Expected columns:

- `lat`
- `lon`
- `elevation_m`

Purpose:

- Computes terrain ruggedness penalty in effective walking distance model.
- Powers the interactive 3D terrain output generated by `src/05_visualise_results.py`.

### `raw/hk_ramps_points.csv`

Expected columns:

- `lat`
- `lon`
- optional tag metadata fields from Overpass extraction

Purpose:

- Computes nearest ramp distance for each micro-grid cell.
- Supports district-level ramp coverage indicator.

## Data Quality and Refresh Notes

- APIs can return schema and content variations over time.
- Route-stop relationships may include temporary service changes.
- OSM-based ramp proxies are community-mapped and not guaranteed complete.
- Elevation sampling is a modeled surface and should not be interpreted as engineering-grade survey data.

## Recommended Refresh Process

1. Run `python src/01_fetch_data.py`.
2. Verify that all expected files are present in `data/raw/`.
3. Re-run the full pipeline to avoid mixed-version outputs.
4. Archive timestamped snapshots for reproducibility when preparing reports.

## Governance and Usage

- Respect original source licensing and attribution obligations.
- Treat this data stack as analytical decision support, not legal or safety-critical ground truth.
